# 《拾光循迹》校园失物招领微信小程序技术说明（最新版）

## 一、项目名称
**拾光循迹——校园失物招领微信小程序**

## 二、项目背景

在高校校园中，学生日常活动频繁，失物遗失和拾取现象较为常见，例如校园卡、钥匙、书本、耳机、充电器、证件等。当前很多学校的失物招领仍主要依赖微信群、朋友圈、宿管处、保卫处或校园墙等渠道，存在以下问题：

1. 信息分散，难以统一管理；
2. 发布方式不规范，检索效率低；
3. 内容更新不及时，容易遗漏；
4. 缺乏统一的状态管理与内容审核机制。

基于此，本项目设计并实现一个面向校园场景的微信小程序，帮助师生便捷发布、浏览、筛选和管理失物招领信息，提高失物归还效率，营造校园互助氛围。

## 三、项目目标

本项目旨在构建一个轻量化、易传播、可扩展的校园失物招领服务平台，主要目标如下：

1. 提供统一的失物与招领信息发布入口；
2. 实现信息浏览、分类筛选、关键词搜索等基础查询能力；
3. 支持用户上传图片、填写地点、时间和联系方式；
4. 实现“发布—查找—联系—完成”的业务闭环；
5. 接入基础自动审核与管理员管理能力，增强平台规范性；
6. 基于微信云开发降低部署门槛，提高项目落地可行性。

## 四、项目定位

本项目定位为：

### 1. 校园场景工具类服务小程序
主要服务对象为：
- 在校学生
- 教师
- 校园管理人员

### 2. 比赛展示与实际落地兼顾的原型系统
项目既满足校园“微创新”比赛的展示要求，又具备后续扩展为真实运行平台的技术基础。

### 3. 公益互助型信息平台
项目强调校园文明、互助归还和信息规范管理。

## 五、核心用户角色

### 1. 失主
需求包括：
- 发布寻物信息
- 浏览是否有人拾到
- 查看物品详情与联系方式
- 联系对方并在找回后更新状态

### 2. 拾到者
需求包括：
- 发布招领信息
- 上传物品图片
- 描述拾取地点与时间
- 联系失主并在归还后标记完成

### 3. 管理员
需求包括：
- 查看全部帖子
- 查看审核状态
- 手动通过 / 驳回帖子
- 删除违规或异常内容

## 六、系统功能模块

### 1. 首页模块
首页主要展示平台入口与最新信息，包括：
- 项目简介
- 快捷入口（我捡到了 / 我丢失了）
- 分类浏览
- 最新信息列表
- 下拉刷新

### 2. 发布模块
支持两类信息发布：
- 招领信息（我捡到了）
- 寻物信息（我丢失了）

发布字段包括：
- 标题
- 物品名称
- 分类
- 地点
- 时间（日期 + 时间选择器）
- 联系方式类型（微信 / QQ / 电话 / 其他）
- 联系方式内容
- 图片上传
- 详细描述

### 3. 列表模块
支持：
- 全部 / 招领 / 寻物 切换
- 分类筛选
- 关键词搜索
- 最新信息高亮
- 图片展示
- 下拉刷新

### 4. 详情模块
展示：
- 标题
- 类型
- 状态标签
- 审核状态
- 时间与分类
- 图片
- 联系信息
- 详细描述

支持：
- 联系信息复制
- 联系发布者
- 标记为已完成
- 删除帖子

### 5. 我的页面
用于展示和维护当前用户信息，包括：
- 头像
- 昵称
- 已发布数量
- 个人资料完善
- 我的发布入口
- 管理中心入口（管理员可见）

### 6. 我的发布模块
支持：
- 查看当前用户发布的帖子
- 按状态筛选（全部 / 进行中 / 已完成）
- 查看审核通过 / 未通过状态
- 查看审核原因

### 7. 管理中心模块
管理员专用，支持：
- 查看全部帖子
- 按审核状态筛选
- 手动通过审核
- 手动驳回审核
- 删除任意帖子
- 删除帖子时同步删除云存储图片

## 七、技术方案

### 1. 前端技术
采用微信小程序原生开发：
- WXML
- WXSS
- JavaScript

### 2. 后端与云服务
采用微信云开发（CloudBase）：
- 云数据库
- 云存储
- 云函数

### 3. 数据存储
核心集合包括：
- `posts`：帖子信息
- `userProfiles`：用户资料
- `admins`：管理员白名单

### 4. 云函数
当前已实现：
- `login`：获取当前用户 openid
- `getTempFileURL`：获取云存储图片临时访问链接
- `checkAdmin`：校验当前用户是否为管理员
- `fixPosts`：修复历史数据字段

## 八、系统架构说明

系统采用“前端小程序 + 云开发服务端”的结构。

### 架构层次
1. **展示层**  
   小程序页面、组件、按钮、表单、列表、详情展示

2. **业务逻辑层**  
   页面逻辑、发布逻辑、筛选逻辑、状态切换、资料保存、审核判断

3. **云服务层**  
   云数据库、云存储、云函数

4. **数据层**  
   帖子、用户资料、管理员名单

## 九、数据库设计

- `openid`

## 十、关键技术实现说明

### 1. 用户身份识别
通过云函数 `login` 获取当前用户 `openid`，实现：
- 区分用户发布内容
- 管理“我的发布”
- 判断详情页是否为本人发布
- 管理员权限判断

### 2. 云端图片上传与展示
用户发布帖子时：
1. 本地选择图片
2. 上传至云存储
3. 数据库存储图片 `fileID`

由于云存储权限限制，项目通过云函数 `getTempFileURL` 获取图片临时可访问链接，从而解决私有读场景下的图片展示问题。

### 3. 自动基础审核
发布内容会经过基础敏感词检测。  
若命中预设风险词，则：
- `auditStatus = rejected`
- `auditRemark = 命中敏感词`

未命中则自动通过。

### 4. 管理员审核与管理
系统通过：
- `admins` 集合存储管理员白名单
- `checkAdmin` 云函数校验管理员身份

实现管理员中心功能，包括：
- 查看全部帖子
- 手动通过 / 驳回
- 删除任意帖子

### 5. 删除联动机制
删除帖子时：
1. 获取对应帖子详情
2. 找出其中的云存储图片
3. 先删除云存储图片
4. 再删除数据库中的帖子记录

从而避免产生无效图片残留。

### 4. 校园公益工具与平台管理结合
既满足校园失物招领的实际需求，又体现系统管理和可扩展能力。
